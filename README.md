# Specter

A basic, **screenshot-protected** internet browser built with [Tauri 2](https://v2.tauri.app/).

The window has content protection enabled (`set_content_protected(true)`), so other
applications — and the OS screen-capture/recording pipeline — cannot capture it:

- **macOS** → `NSWindowSharingType::None` (screenshots & screen recordings come out blank)
- **Windows** → `WDA_EXCLUDEFROMCAPTURE`

## How it works

A single native window hosts a **toolbar webview** plus **one content webview per tab**
(Tauri 2's multi-webview feature, behind the `unstable` Cargo feature):

| Webview   | Source                | Role                                              |
| --------- | --------------------- | ------------------------------------------------- |
| `toolbar` | local `ui/index.html` | The chrome: tab strip + back / forward / reload / home + address bar |
| `tab-N`   | external URLs         | One per open tab; renders the actual web pages    |

The toolbar calls Rust commands (`navigate`, `go_back`, `go_forward`, `reload`, `go_home`,
`new_tab`, `select_tab`, `remove_tab`) via `window.__TAURI__.core.invoke`. Rust tracks the
open tabs (`TabState`) and drives whichever tab is **active**; the active tab fills the
content area while the others are parked off-screen (kept alive in the background). As the
active tab navigates, a `url-changed` event syncs the address bar and a `tabs-changed` event
redraws the tab strip. A window-resize handler re-lays-out the toolbar and tabs.

**Shortcuts:** `Cmd/Ctrl+T` new tab · `Cmd/Ctrl+W` close tab · `Cmd/Ctrl+Shift+W` close window ·
`Cmd/Ctrl+L` focus address bar.

Using a real native webview for content (instead of an `<iframe>`) means sites that send
`X-Frame-Options` / frame-busting CSP (Google, GitHub, etc.) load normally.

## Project layout

```
khufia/
├── ui/index.html              # toolbar UI (no bundler — plain HTML/JS/CSS)
├── generate-icon.js           # regenerates app-icon.png (no deps)
├── app-icon.png               # icon source for `tauri icon`
└── src-tauri/
    ├── Cargo.toml             # tauri = { features = ["unstable"] }
    ├── tauri.conf.json        # frontendDist = ../ui, withGlobalTauri = true
    ├── capabilities/default.json
    ├── icons/                 # generated by `npm run tauri icon`
    └── src/main.rs            # window + 2 webviews + content protection + commands
```

## Run it

Prerequisites: Rust toolchain and the [Tauri system dependencies](https://v2.tauri.app/start/prerequisites/).
The Tauri CLI is provided as a dev dependency in `package.json`.

```bash
npm install        # installs @tauri-apps/cli (v2)
npm run dev        # launches the browser in dev mode
npm run build      # produces a release bundle
```

(You can also run the Rust app directly: `cd src-tauri && cargo run`.)

## Regenerating the icon

```bash
node generate-icon.js                  # writes app-icon.png
npm run tauri icon app-icon.png        # regenerates src-tauri/icons/*
```

## Notes & caveats

- Content protection blocks capture by other software, but **does not** stop someone
  photographing the screen with a separate camera.
- Multi-webview relies on Tauri's `unstable` feature, so APIs (`Window::add_child`,
  `Manager::get_webview`) may change in future Tauri releases.
- The default home page / search engine is Google; change `HOME_URL` in
  `src-tauri/src/main.rs`.
